Interpretation:
- OpenVINS is now a usable VIO trajectory source for this session, especially with noisy IMU + ZUPT.
- OpenVINS does not provide a full reusable SLAM map; its SLAM points are sparse filter landmarks.
- ORB-SLAM3 is the right next target for a reusable sparse atlas, but the current custom run is not stable yet.

Recommended next technical routes:
- Patch/build ORB-SLAM3 from source with better logging and a safer reset path instead of using the opaque Docker binary.
- Try ORB-SLAM3 mono-inertial with a dataset format that uses actual image timestamps rather than the gopro_slam wrapper's frame_idx/fps timing.
- Use the OpenVINS ZUPT trajectory as the trusted pose prior and build a separate visual map/point cloud offline.
- For dense metric mapping, collect stereo/RGB-D/LiDAR data; monocular-inertial alone is mainly sparse SLAM.
